# 백엔드 파일 분리 작업 공유

작업일: 2026-06-09 | 작업자: 송지현

---

## 작업 배경

`backend_main.py` 하나에 앱 설정 + 채팅 비즈니스 로직이 전부 들어있어서 파일이 너무 비대했음.
역할별로 파일을 나눠서 유지보수가 쉽게 구조를 정리함.

---

## 변경된 파일 구조

```
backend/
├── backend_main.py          ← 앱 설정 + 라우터 연결만 남김 (정리 완료)
├── routers/
│   ├── auth.py              ← 변경 없음
│   ├── counseling.py        ← /chat 라우터 추가, import 경로 수정
│   ├── survey.py            ← 나중에 추가 예정
│   └── guardian.py          ← 나중에 추가 예정
├── services/
│   ├── chat_service.py      ← 신규 생성 (채팅 비즈니스 로직)
│   ├── audit_service.py     ← 신규 생성 (감사 로그 헬퍼)
│   └── summary_service.py   ← 위치 이동 (backend/ → services/)
└── core/
    ├── crypto.py            ← 변경 없음
    └── security.py          ← 변경 없음
```

---

## 각 파일 역할

| 파일 | 역할 |
|---|---|
| `backend_main.py` | FastAPI 앱 생성, DB 테이블 초기화, 라우터 연결만 |
| `routers/counseling.py` | 상담 관련 URL 처리 (/counseling/chat 포함) |
| `services/chat_service.py` | 채팅 비즈니스 로직 전담 (AI 호출, 메시지 저장 등) |
| `services/audit_service.py` | 감사 로그 저장 헬퍼 함수 (log_sensitive, log_general) |
| `services/summary_service.py` | LoRA 요약 서버 호출 |

## chat_service.py 내부 구조

```
process_chat() 함수
├── 1. JWT 토큰 검증
├── 2. 세션 조회/생성
├── 3. 60분 타임아웃 체크 (자동 세션 종료 + 재상담 전환)
├── 4. system_prompt 결정
│   ├── 캐시 있으면 → 재사용
│   ├── 이전 요약 있으면 → Agent 호출 (재상담)
│   └── 첫 상담이면 → general_prompt 사용 (외국어 버그 수정)
├── 5. Groq 응답 요청
├── 6. 외국어 감지 시 재요청
├── 7. 메시지 저장 (암호화)
└── 8. 감사 로그 저장
```

---

## 추가로 수정된 사항

### 외국어 버그 수정
기존에 첫 상담 시작할 때 system_prompt가 빈 문자열이어서 AI가 영어로 답하는 버그가 있었음.
`general_prompt.txt` 를 연결해서 첫 상담에도 한국어 지침이 들어가도록 수정함.

### summary_service.py import 경로 수정
`counseling.py` 에서 `from summary_service` → `from services.summary_service` 로 변경.
