# [프론트 요청서] 채팅 접속량 초과(429) 안내 문구 처리

## 배경 / 목적
LLM(Cerebras) 추론 서버가 트래픽 초과로 `429 (queue_exceeded)`를 반환하는 경우가 있습니다.
지금까지는 백엔드 `/counseling/chat`이 이 예외를 잡지 않아 **500 Internal Server Error**로 떨어졌고,
프론트는 모든 비정상 응답을 동일한 폴백 문구(`"현재 서버에 연결할 수 없어요. 잠시 후 다시 시도해주세요."`)로 표시했습니다.

이번에 **백엔드를 수정**해서, LLM 접속량 초과인 경우에는 500이 아니라 **HTTP 429**로 명확히 구분해서 내려줍니다.
프론트에서 이 429를 받아 **"접속량이 많습니다. 잠시 후에 시도해주세요."** 안내를 노출해 주세요.

## 백엔드 변경 사항 (완료 · 참고용)
- 파일: `backend/routers/counseling.py` → `POST /counseling/chat`
- LLM 호출이 `RateLimitError`를 던지면 아래 형태로 응답합니다.

```
HTTP/1.1 429 Too Many Requests
Content-Type: application/json

{ "detail": "접속량이 많습니다. 잠시 후에 시도해주세요." }
```

- 그 외 일반 오류(코드 버그·DB 오류 등)는 기존과 동일하게 500으로 내려갑니다. (429와 구분됨)

## 요청 사항 (프론트)
`POST /counseling/chat` 응답에서 **`status_code == 429`인 경우를 별도 분기**로 처리해 주세요.

### 대상 코드 (현재)
`frontend/app/services/api_client.py` → `send_chat_message()` (약 98~164줄)
지금은 `res.ok`가 아니고 401도 아니면 전부 맨 끝 폴백(164줄)으로 떨어집니다.

```python
        if res.ok:
            return res.json().get("reply", "응답을 받지 못했어요.")
        # 401이면 refresh 시도
        if res.status_code == 401:
            ...
    except requests.RequestException:
        pass
    return "현재 서버에 연결할 수 없어요. 잠시 후 다시 시도해주세요."   # ← 모든 비정상 응답 공통 폴백
```

### 변경안 (권장)
`res.ok` 체크 바로 다음, 401 분기 앞쪽에 429 분기를 추가:

```python
        if res.ok:
            return res.json().get("reply", "응답을 받지 못했어요.")

        # 접속량 초과 → 백엔드가 429로 내려줌
        if res.status_code == 429:
            return "접속량이 많습니다. 잠시 후에 시도해주세요."

        # 401이면 refresh 시도
        if res.status_code == 401:
            ...
```

- 맨 끝 164줄 폴백(`"현재 서버에 연결할 수 없어요..."`)은 **그대로 유지**해 주세요. 429 외 다른 오류용입니다.
- 백엔드가 보내는 `detail` 문구를 그대로 쓰고 싶으면 `res.json().get("detail", "접속량이 많습니다. 잠시 후에 시도해주세요.")`로 받아도 됩니다. (문구 동기화가 부담되면 위처럼 프론트에 하드코딩해도 무방)

## 참고 / 주의
- 현재 구조상 이 함수가 반환한 문자열은 채팅창에 **어시스턴트 말풍선**으로 표시됩니다. (기존 폴백 문구와 동일한 표시 방식)
  - 별도 토스트/배너로 구분 노출하고 싶다면 `chat.py` 라우트(`room()`)에서 분기 처리가 추가로 필요합니다. 우선은 말풍선 노출만으로 충분하면 위 변경만으로 됩니다.
- `room.html`의 fetch 에러 문구(`답변을 불러오지 못했어요...`, `네트워크 연결이 불안정해요...`)는 **Flask 라우트가 비정상일 때**용이라 이번 429와는 별개입니다. 수정 불필요.

## 요약
백엔드: LLM 429 → `/counseling/chat`이 `HTTP 429 + detail` 반환 (완료).
프론트: `send_chat_message()`에 `status_code == 429` 분기 1개 추가 → "접속량이 많습니다. 잠시 후에 시도해주세요." 노출.
